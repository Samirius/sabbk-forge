# Brand Playbook (identity)

> Brand strategy: define → document → enforce → evolve.

## Phases

### Phase 1: SCAN
- Read all brand touchpoints (website, social, docs, proposals)
- Audit consistency across channels
- Map brand perception (what we say vs what people see)
- Output: `brand-context.json` + consistency report + perception map

### Phase 2: DEFINE
- Codify brand identity (mission, vision, values, personality)
- Define visual identity (logo usage, colors, typography)
- Set tone of voice guidelines
- Output: brand guide document

### Phase 3: DOCUMENT
- Create brand asset library (templates, guidelines, examples)
- Build brand playbook (do's and don'ts, usage rules)
- Generate AGENTS.md brand rules for other pi agents
- Output: complete brand documentation package

### Phase 4: ENFORCE
- All pi agents check outputs against brand rules
- Automated brand consistency scoring
- Flag violations for human review
- Output: brand compliance reports

### Phase 5: EVOLVE
- Track brand performance over time
- Propose updates based on market changes
- Version brand guidelines with changelog
- Output: updated brand documentation

## Quality Gates
- Brand guide completeness (all sections filled)
- Cross-channel consistency (visual + tone)
- Agent compliance (all pi agents enforce brand rules)
- Version control (all changes tracked)

## Tool: `identity`
```
identity scan <brand>      → audit current state
identity define <brand>    → codify brand identity
identity document <brand>  → create brand guide
identity enforce <brand>   → compliance check
identity evolve <brand>    → track + update
```
